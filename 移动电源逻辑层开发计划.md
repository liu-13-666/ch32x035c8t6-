# 移动电源逻辑层开发计划

## 目标

基于 CH32X035C8T6 做移动电源比赛作品。你负责逻辑层，把底层 AD、PD、I2C/OLED 和后续 GPIO 控制接口组合成完整产品行为。

第一版目标是稳定演示：插入 PD 电源识别、充电状态、放电状态、保护状态、OLED 展示和串口调试。

## 电气规格目标

- 输入支持：`9V/2A` 或 `5V/2A`。
- 输入策略：优先向 PD 适配器申请 `9V/2A`，如果适配器不支持，再退回 `5V/2A`。
- 输出目标：`5V/2A`。
- 输出过流保护：暂定 `2300mA`，略高于 2A，避免正常瞬态误触发。
- 保护策略覆盖：输入异常、输出过流、输出短路、电池低压/过压、温度异常。

## 当前实现

- `main.c` 已初始化 AD、PD、I2C、OLED、Power Manager。
- 主循环每 1ms 调用 `PDSink_Task()`。
- 主循环每 100ms 调用 `Power_Manager_Task()` 和 `UI_Task()`。
- 心跳灯使用 PC16，避免和 OLED 的 PA0/PA1 软件 I2C 冲突。
- `board_power_port.c` 负责底层适配，把逻辑层接口映射到已有 AD/PD 函数。
- `PDSink.c` 已实现输入档位自动选择：优先 `9V/2A`，其次 `5V/2A`，最后兜底第一个可用 Fixed PDO。
- OLED 显示状态、SOC、电池电压、温度、输出电压电流、PD 状态、已协商输入电压和故障码。

## 状态机策略

- `PM_STATE_INIT`：上电初始化状态，随后进入空闲。
- `PM_STATE_IDLE`：空闲，等待 PD 输入或负载接入。
- `PM_STATE_CHARGING`：PD ready 后进入充电，第一版关闭输出。
- `PM_STATE_DISCHARGING`：检测到负载后进入放电，第一版关闭充电。
- `PM_STATE_PROTECT`：低压、过压、过流、过温时关闭充电和输出。

第一版不做边充边放。输入和输出同时存在时，优先进入充电，方便比赛现场稳定演示。

## 保护阈值

- 电池低压：`3300mV`
- 低压恢复：`3400mV`
- 电池过压：`4200mV`
- 满电退出充电：`4180mV`
- 输出目标：`5000mV / 2000mA`
- 输出过流：`2300mA`
- 温度范围：`0C` 到 `60C`
- 负载检测：输出电流不低于 `50mA`

这些值是第一版演示阈值，后续要结合电池规格、采样电阻、分压比例和真实负载校准。

## 后续必须确认

- 充电使能引脚，或者充电芯片 I2C 命令。
- 输出/升压/MOS 使能引脚。
- 负载检测是只用输出电流，还是有专门检测脚。
- 电池电压分压比例是否已经在 AD 层换算。
- 电流采样电阻真实阻值是否和代码假设一致。
- NTC 型号和上下拉电阻是否与代码参数一致。
- 9V 输入路径是否经过充电芯片或降压电路允许，不能直接把 9V 加到只支持 5V 的器件上。

## OLED 显示说明

示例：

```text
ST:CHG    S075
B3700mV T+025C
O5000mV 1200mA
PD:READY I9.0F00
```

- `ST`：当前电源状态。
- `S`：粗略电量百分比 SOC。
- `B`：电池电压。
- `T`：温度；如果没有确认 NTC，这个数只作占位显示。
- `O`：5V 输出电压和输出电流。
- `PD`：PD 协商状态。
- `I9.0` / `I5.0`：已协商到的输入电压。
- `F00`：故障码。`F` 是 Fault，不是 FOD。

故障码：

- `F00`：无故障
- `F01`：电池低压
- `F02`：电池过压
- `F04`：输出过流
- `F08`：温度异常

PD 状态：

- `DISC`：未检测到 Type-C/PD 输入。
- `CONN`：CC 已连接，等待 Source Cap。
- `NEGO`：已发送 Request，等待 ACCEPT/PS_RDY。
- `READY`：已收到 PS_RDY，PD 输入协商完成。
- `REJ`：适配器拒绝请求。
- `WAIT`：适配器要求等待。
- `TXF`：Request 发送失败，通常是没有收到 GoodCRC。
- `ACTO`：等待 ACCEPT 超时。
- `PSTO`：等待 PS_RDY 超时。

## 外部中断说明

普通“外部中断”一般指 GPIO 引脚边沿触发，例如按键按下后打断主循环。

本项目的 PD 不是普通 GPIO 外部中断，而是 USBPD 外设中断。函数是：

```c
void USBPD_IRQHandler(void)
```

USBPD 外设收到 PD 包、发送结束或收到 Reset 时，会打断主循环进入这个函数。这样 MCU 不需要一直死等 PD 包，PD 收发事件来了就能及时处理。

## 调试提醒

如果先插 Type-C 再给板子上电，适配器可能已经发过第一轮 Source Cap。板子上电后可能只显示 `CONN`，但重新插拔 Type-C 后又能 `READY`。

比赛演示建议：

1. 先给板子上电。
2. 等 OLED 正常显示。
3. 再插 Type-C PD 电源。
4. 观察 OLED 从 `CONN`、`NEGO` 到 `READY`。

## 测试清单

- 上电后 OLED 能显示 `INIT -> IDLE`，串口能打印状态变化。
- 插入支持 9V/2A 的 PD 电源后，OLED 第 4 行显示已协商输入约 `9.0V`。
- 插入只支持 5V/2A 的 PD 电源后，OLED 第 4 行显示已协商输入约 `5.0V`。
- PD 状态到 `READY` 后，状态机进入充电。
- 拔出 PD 电源后，状态机回到空闲。
- 接入 5V 输出负载后，状态机进入放电。
- 输出电流超过 `2300mA` 时进入保护。
- 电池低压、过压、温度异常时进入保护。
- 保护状态下充电和输出都关闭。
- 异常解除后回到空闲。
